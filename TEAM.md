# MediChat-RD 多Agent协作团队

## 团队架构

```
                    ┌──────────────────┐
                    │   项目经理 (PM)   │
                    │   协调 & 追踪     │
                    └────────┬─────────┘
                             │
        ┌────────┬───────────┼───────────┬────────┐
        ▼        ▼           ▼           ▼        ▼
  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
  │ 罕见病   │ │ 架构师   │ │ 编程专家 │ │ 质控专家 │ │ 前端专家 │
  │ 专家     │ │          │ │ (后端)   │ │ (QC)     │ │          │
  └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘
```

## 角色定义

### 1. 罕见病专家 (Rare Disease Expert)
- **职责**：疾病知识库设计、诊疗流程规范、医学术语标准化
- **产出**：疾病数据结构、症状关联图谱、诊疗指南
- **对标**：NORD/Orphanet/CHARD

### 2. 架构师 (System Architect)
- **职责**：系统架构设计、技术选型、API规范、数据库设计
- **产出**：架构文档、API规范、数据模型
- **技术栈**：FastAPI + React + PostgreSQL + Milvus + CrewAI

### 3. 编程专家 (Backend Developer)
- **职责**：后端API开发、Agent实现、数据库操作
- **产出**：API代码、Agent逻辑、数据库迁移

### 4. 前端专家 (Frontend Developer)
- **职责**：UI组件开发、页面实现、交互优化
- **产出**：React组件、页面、样式

### 5. 质控专家 (QC Engineer)
- **职责**：代码审查、测试用例、安全检查、性能优化
- **产出**：审查报告、测试结果、修复建议

### 6. 项目经理 (Project Manager)
- **职责**：任务分配、进度跟踪、风险管控、资源协调
- **产出**：进度报告、任务清单、里程碑

## 协作流程

1. PM 拆解任务 → 分配给对应专家
2. 各专家并行工作 → 提交产出
3. QC 审查产出 → 反馈问题
4. PM 汇总进度 → 汇报给小林医生
