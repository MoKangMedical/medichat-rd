## 🔧 核心模块

### 1. 多智能体诊断引擎
- **分诊Agent**：症状收集、SOAP格式结构化
- **诊断Agent**：OMIM/HPO知识图谱驱动的鉴别诊断
- **基因Agent**：ClinVar变异解读、基因型-表型关联
- **药物Agent**：药物重定位、靶点-疾病网络
- **循证Agent**：PubMed实时检索、证据分级
- **防幻觉守卫**：三层校验（知识图谱→文献→共识）

### 2. 知识库
- **OMIM**：25,000+基因疾病关联
- **HPO**：16,000+人类表型本体
- **Orphanet**：6,000+罕见病数据库
- **PubMed**：36M+生物医学文献
- **自建知识图谱**：200+罕见病详细诊疗路径

### 3. MIMO集成
- 接入小米MIMO API作为核心对话引擎
- 支持长上下文（1M tokens）—— 病史越完整，诊断越精准
- 结构化提示词工程，引导AI按诊疗路径思考

## 🚀 快速开始

### 本地部署

```bash
# 克隆仓库
git clone https://github.com/MoKangMedical/medichat-rd.git
cd medichat-rd

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入 MIMO_API_KEY

# 启动服务
bash scripts/deploy.sh
```

### 访问
- Web界面：http://localhost:8080
- API文档：http://localhost:8080/docs

## 🔬 Harness理论

MediChat-RD 的竞争优势不在于底层大模型（我们用MIMO API），而在于**诊断Harness**设计：

```
诊断Harness = 症状解析器 + 知识图谱检索 + 多Agent编排 + 防幻觉校验 + 报告生成
```

- 模型是通用的，Harness是私有的
- 越好的Harness → 越深的护城河
- 我们的核心资产不是模型权重，是诊断流程的知识编码
