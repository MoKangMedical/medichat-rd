# MEDVi模式 × MediChat-RD：中国版罕见病远程医疗平台

> 借鉴Medvi"极简架构"模式，适配中国医疗法规和市场环境

## 核心理念

```
Medvi模式：只做流量层，医疗合规全外包
我们的模式：只做AI+流量层，医疗合规全外包+技术壁垒加护
```

## 架构对比

| 层 | Medvi (美国) | MediChat-RD (中国) |
|---|---|---|
| **流量层** | 付费广告(Google/Meta) | 微信生态(公众号/小程序/朋友圈) |
| **品牌层** | medvi.org独立站 | 微信公众号+小程序+抖音 |
| **AI层** | ChatGPT/Claude/Grok | MIMO API无限额度+66个Skill |
| **医疗层** | CareValidate+OpenLoop(全外包) | 合作医院+互联网医院牌照(半外包) |
| **药品层** | 合成GLP-1药房(全外包) | 罕见病特药+海外药房(合作模式) |
| **合规层** | FDA灰色地带 | 国家卫健委+医保局合规框架 |

## 中国适配要点

### 1. 医疗合规
- 必须依托持牌互联网医院
- 医生必须有执业资格并在合作机构注册
- 处方药必须通过合规药房

### 2. 流量获取
- 微信生态：公众号内容+小程序+社群
- 抖音/小红书：科普内容引流
- 患者社群：罕见病组织合作

### 3. 技术壁垒（Medvi没有的）
- AI辅助罕见病诊断（MIMO API驱动）
- 多源知识库整合（PubMed/OMIM/ChEMBL）
- 患者画像+个性化治疗方案
- 长期随访管理系统
